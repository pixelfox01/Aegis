# Docker Compose Configuration

Aegis provides two deployment modes via Docker Compose.

## Deployment Modes

### Combined Deployment (Recommended)

Single container serving both frontend and backend on port 8000.

**Pros:**
- Simpler setup
- Single port to expose
- Lower resource usage

**Cons:**
- Less flexibility
- Cannot scale frontend/backend independently

### Separate Deployment

Frontend and backend in separate containers.

**Pros:**
- Can scale independently
- More flexible configuration
- Better for production at scale

**Cons:**
- More complex setup
- Need to expose multiple ports

## Combined Deployment

Use the default `docker-compose.yml`:

```yaml
services:
  aegis:
    build: .
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=postgresql://aegis:${POSTGRES_PASSWORD}@db:5432/aegis
      - JWT_SECRET_KEY=${JWT_SECRET_KEY}
      - AUTH_MODE=${AUTH_MODE:-local}
      - LLM_PROVIDER=${LLM_PROVIDER:-gemini}
      - GEMINI_API_KEY=${GEMINI_API_KEY}
      - OLLAMA_BASE_URL=${OLLAMA_BASE_URL:-http://ollama:11434}
      - OLLAMA_MODEL=${OLLAMA_MODEL:-llama2}
      - CORS_ORIGINS=${CORS_ORIGINS:-http://localhost:8000}
    depends_on:
      db:
        condition: service_healthy
    restart: unless-stopped

  db:
    image: postgres:16-alpine
    environment:
      - POSTGRES_USER=aegis
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD}
      - POSTGRES_DB=aegis
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U aegis"]
      interval: 10s
      timeout: 5s
      retries: 5
    restart: unless-stopped

volumes:
  postgres_data:
```

Start with:

```bash
docker-compose up -d
```

Access at: **http://localhost:8000**

## Separate Deployment

Use `docker-compose.separate.yml`:

```yaml
services:
  frontend:
    build:
      context: ./frontend
      args:
        - VITE_API_URL=http://localhost:8000
        - VITE_AUTH_MODE=${AUTH_MODE:-local}
    ports:
      - "3000:3000"
    depends_on:
      - backend
    restart: unless-stopped

  backend:
    build: ./backend
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=postgresql://aegis:${POSTGRES_PASSWORD}@db:5432/aegis
      - JWT_SECRET_KEY=${JWT_SECRET_KEY}
      - AUTH_MODE=${AUTH_MODE:-local}
      - LLM_PROVIDER=${LLM_PROVIDER:-gemini}
      - GEMINI_API_KEY=${GEMINI_API_KEY}
      - CORS_ORIGINS=${CORS_ORIGINS:-http://localhost:3000}
    depends_on:
      db:
        condition: service_healthy
    restart: unless-stopped

  db:
    image: postgres:16-alpine
    environment:
      - POSTGRES_USER=aegis
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD}
      - POSTGRES_DB=aegis
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U aegis"]
      interval: 10s
      timeout: 5s
      retries: 5
    restart: unless-stopped

volumes:
  postgres_data:
```

Start with:

```bash
docker-compose -f docker-compose.separate.yml up -d
```

Access at: **http://localhost:3000**

## Adding Ollama

To use Ollama for local LLM processing, add this service:

```yaml
services:
  ollama:
    image: ollama/ollama:latest
    ports:
      - "11434:11434"
    volumes:
      - ollama_data:/root/.ollama
    restart: unless-stopped

volumes:
  ollama_data:
```

Then set in your `.env`:

```bash
LLM_PROVIDER=ollama
OLLAMA_BASE_URL=http://ollama:11434
OLLAMA_MODEL=llama2
```

Pull a model:

```bash
docker exec -it aegis-ollama-1 ollama pull llama2
```

## Customization

### Change Ports

```yaml
services:
  aegis:
    ports:
      - "8080:8000"  # External:Internal
```

### Add Volume Mounts

For persistent file storage:

```yaml
services:
  aegis:
    volumes:
      - ./data:/app/data
```

### Resource Limits

```yaml
services:
  aegis:
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 2G
```

## Maintenance

### View Logs

```bash
docker-compose logs -f
docker-compose logs -f aegis  # specific service
```

### Restart Services

```bash
docker-compose restart
```

### Update Images

```bash
docker-compose pull
docker-compose up -d
```

### Backup Database

```bash
docker exec aegis-db-1 pg_dump -U aegis aegis > backup.sql
```
